# Logging in Kubernetes

## What is Logging in Kubernetes?

Logging in Kubernetes refers to the process of **collecting, viewing, and managing logs generated by containers running inside Pods**.

Logs help you:
- Debug applications  
- Monitor behavior  
- Troubleshoot failures  

---

## Docker vs Kubernetes Logs

### Docker
```docker logs -f <container-id>```

Streams logs directly from a container.


### Kubernetes
```kubectl logs -f <pod-name>```

Streams logs from a container inside a Pod

## Example: Single Container Pod
```
apiVersion: v1
kind: Pod
metadata:
    name: event-simulator-pod
spec:
    containers:
    - name: event-simulator
      image: mycloud/event-simulator
```

### Create the Pod
```kubectl create -f event-simulator.yaml```

### View Logs
```kubectl logs -f event-simulator-pod```

Explanation:

* `logs` -> fetch logs
* `-f` -> follow (stream logs in real time)

## Example: Multi-Container Pods

When a Pod has multiple containers, you must specify which container’s logs you want.

```
apiVersion: v1
kind: Pod
metadata:
    name: event-simulator-pod
spec:
    containers:
    - name: event-simulator
      image: mycloud/event-simulator
    - name: image-processor
      image: image-processor
```

### View Logs for Specific Container
```kubectl logs -f event-simulator-pod event-simulator```

Explanation:

* event-simulator-pod -> Pod name
* event-simulator -> container name



## Important Notes
**Logs are per container**
* Each container has its own log stream

**Logs are ephemeral**
* If a Pod is deleted -> logs are lost
* If a container restarts -> old logs may be lost unless persisted

**View logs of a crashed container**

```kubectl logs <pod-name> --previous```
* Shows logs from the previous container instance


## Advanced Logging (Production)

In real-world systems, logs are not stored only in Pods.

Instead, use centralized logging systems:

- ELK Stack (Elasticsearch, Logstash, Kibana)
- Fluentd / Fluent Bit
- Cloud logging (AWS CloudWatch, GCP Logging)


## Key Takeaways
* `kubectl logs` = primary debugging tool
* `-f` = follow logs in real-time
* Multi-container Pods -> specify container name
* Logs are ephemeral by default
* Production -> use centralized logging
